# AI自动化工作流辅助系统 - 项目文档纪要

## 项目概述

### 项目背景
本项目旨在搭建一个AI自动化工作流辅助系统的基础框架，为影视创作、广告制作等场景提供智能化的剧本生成、文生图、图生视频和素材整理功能。

### 核心功能模块
1. **剧本生成** - 基于故事信息和前提条件生成完整剧本
2. **文生图** - 根据文本描述生成对应的图像素材
3. **图生视频** - 基于图像生成短视频内容
4. **素材整理** - 智能分类和管理创作素材

## 技术架构

### 后端技术栈
- **框架**: Spring Boot 3.x
- **ORM**: MyBatis
- **数据库**: MySQL
- **缓存**: Redis
- **构建工具**: Maven
- **API文档**: Swagger/OpenAPI

### 前端技术栈
- **框架**: Vue 3 + Composition API
- **UI组件库**: Element Plus
- **构建工具**: Vite 4.3.9
- **包管理**: npm

### 系统架构特点
- **前后端分离** - 清晰的职责划分，便于独立开发和部署
- **松耦合设计** - 各功能模块独立，便于后期扩展
- **RESTful API** - 标准化的接口设计
- **组件化开发** - 前端采用组件化，后端采用分层架构

```java
public Map<String, Object> callAIAPI(String storyInfo, String premise, String apiType) {
    // 根据API类型获取配置
    APIConfig config = getAPIConfig(apiType);
    if (config == null) {
        throw new IllegalArgumentException("不支持的API类型: " + apiType);
    }
    
    // 构建提示词和请求参数
    String prompt = buildPrompt(storyInfo, premise, apiType);
    Map<String, Object> requestBody = buildRequestBody(config.getModel(), prompt);
    
    // 发送API请求并处理响应
    try {
        ResponseEntity<Map> responseEntity = restTemplate.exchange(
                config.getApiUrl(),
                HttpMethod.POST,
                new HttpEntity<>(requestBody, buildHeaders(config.getApiKey())),
                Map.class
        );
        return responseEntity.getBody();
    } catch (Exception e) {
        // 返回详细的错误信息便于排查
        Map<String, Object> errorResponse = new HashMap<>();
        errorResponse.put("error", "API调用失败");
        errorResponse.put("message", e.getMessage());
        return errorResponse;
    }
}
```

#### 3. RestTemplateConfig - HTTP客户端配置
- 配置RestTemplate Bean
- 设置连接超时和读取超时
- 支持HTTPS请求

### 1. API调用整合
**问题**：豆包和DeepSeek API调用代码重复，维护困难
**解决方案**：
- 创建统一的API调用方法`callAIAPI`
- 通过配置管理不同API的参数
- 统一的错误处理机制

### 2. 前后端交互优化
**问题**：API调用失败时返回模拟数据不利于问题排查
**解决方案**：
- 返回真实的错误信息
